13 /*
14  *#####PINOUT#####
15  *
16  * OUTPUT 0 - Pin 15
17  * OUTPUT 1 - Pin 1
18  * OUTPUT 2 - Pin 2
19  * OUTPUT 3 - Pin 3
20  * OUTPUT 4 - Pin 4
21  * OUTPUT 5 - Pin 5
22  * OUTPUT 6 - Pin 6
23  * OUTPUT 7 - Pin 7s
24  *
25  * Serial data in (MOSI equiv) - Pin 14
26  * Serial data out - Pin 9
27  * (data out to next chip)
28  * Serial clock - Pin 11
29  *
30  * Latch register - Pin 12
31  *
32  * Latch pin clocks internal register
33  * data to output pins
34  *
35  * Output enable (active low) - Pin 13
36  *
37  * Shift register clear - Pin 10
38  * (active low)
39  *
40  * How chip works:
41  * Data is clocked in using serial data in and
42  * serial clock (on falling edge) then the internal
43  * storage register is clocked to the output pins
44  * once the latch pin is clocked high then low
45  * (rising edge).
46  *
47  * The internal storage register overflows out of
48  * the serial out pin to allow for daisy chaining
49  * of chips.
50  *
51  * #####HOW MY CODE WORKS#####
52  * I have implemented my code in two styles,
53  * one uses a simple number of functions that can
54  * be used to use a status register attached to
55  * the pins specified in the #defines or it can
56  * be used with the shift_array_t object. The
57  * object code allows for multiple shift arrays to
58  * be easily connect to and managed from a single
59  * uController.
60  *
61  * To use the struct obj, set the pins and ports of
62  * the object then call SN54HC595_init_obj as seen
63  * in the example.
64  */
